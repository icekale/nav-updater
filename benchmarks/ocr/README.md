# OCR 基准集

该目录只保存标签模板，不保存净值截图或真实研究数据。真实标签文件建议放在受控本地目录或 Unraid 持久卷中。

## 样本规则

首批选择十份周报中的每份三条产品行，共 30 条。每份至少覆盖以下一种情况：

- 数值完整的产品行；
- 报告用 `-`、`--` 或破折号表示不可用字段的产品行；
- 曾出现名称截断、OCR 易混淆或列错位的产品行。

每条必须填写全部 12 个字段。百分比按照小数写入，例如截图中的 `1.23%` 填为 `"0.0123"`；夏普填写普通数值。只有截图中明确显示不可用占位符的字段填 `null`，看不清或没有确认的字段不能猜测。

## 创建标签

复制示例文件，在 `image` 中填写相对于图片根目录的文件名，在 `sha256` 中填原图指纹：

```bash
shasum -a 256 /path/to/weekly-report.png
```

同一张截图可对应多条产品标签，但每条的 `image` 和 `sha256` 必须一致。不要把 OCR 预测值直接当作标准答案；请以截图中人工核对的值为准。

## 运行

```bash
python scripts/run_ocr_benchmark.py \
  --labels /secure/ocr-labels.json \
  --images-root /secure/weekly-reports \
  --output-dir /secure/ocr-results/2026-07-20
```

命令先校验所有图片 SHA-256，再对每张图片仅识别一次。输出目录必须为空，完成后会生成：

- `summary.md`：产品匹配率、字段准确率、漏识别率和错列率；
- `details.json`：逐产品、逐字段的结果，便于分析错例。

它不会修改数据库、处理历史、Excel 或源图片。
