# 投研净值更新工具

面向投研团队的局域网净值更新工具。它读取固定格式的净值 Excel 模板，结合净值截图和公募基金公开数据，计算收益与风险指标，生成一份新的 Excel，并保存每次更新的历史记录。

这是一个支持 Docker Compose 快速部署的局域网 MVP，适合在同一局域网内由多人共享使用。

## 功能概览

- 上传原始净值 Excel 和一张或多张高清净值截图
- 自动分段识别超长截图，支持一张图同时包含公募和私募产品
- 按产品代码或产品名称进行精确匹配，不做模糊匹配
- 截图缺失的唯一公募名称可自动补全基金代码并查询东方财富公开净值数据
- 私募/渠道产品使用截图 OCR 识别净值指标
- 在人工审核工作台中确认产品、补录指标并记录审核依据
- 将资本市场核心会议 Excel 导入为可检索的会议库，并维护团队跟踪记录
- 计算周收益、月初至今、年初至今、年度收益、Sharpe 和最大回撤
- 保留原始 Excel，生成独立的结果文件
- 保存产品目录、处理批次、文件校验值、处理状态和审计记录
- 支持管理员账号和普通用户账号
- PostgreSQL、上传文件和结果文件使用 Docker 持久卷保存

## 当前边界

以下内容暂不属于当前版本：

- AI 投资分析或自动生成研报
- 微信/企业微信提醒
- 公网部署、HTTPS 证书和自助注册

私募平台没有统一公开接口，工具不会自动登录或绕过平台权限。截图无法识别时，系统会保留原单元格并标红。

## 快速开始

### 环境要求

- Docker Engine
- Docker Compose v2
- 部署主机与使用者在同一局域网

### 配置并启动

```bash
cp .env.example .env
openssl rand -hex 32
```

把生成的随机字符串写入 `.env` 的 `SESSION_SECRET`，并在首次启动前修改管理员账号和密码：

```dotenv
INITIAL_ADMIN_USERNAME=admin
INITIAL_ADMIN_PASSWORD=请替换为强密码
```

启动三个服务：

```bash
docker compose up -d --build
docker compose ps
```

浏览器访问：

```text
http://<部署主机局域网IP>:8080
```

例如部署主机地址为 `192.168.5.241`，则访问 `http://192.168.5.241:8080`。

`app` 容器负责数据库迁移和网页服务，`worker` 容器负责处理任务，`db` 容器运行 PostgreSQL。数据库、上传文件和输出文件都保存在 Docker volume 中，重启容器不会清空历史。

## 首次使用

### 1. 登录

使用 `.env` 中的 `INITIAL_ADMIN_USERNAME` 和 `INITIAL_ADMIN_PASSWORD` 登录。默认示例账号是 `admin / change-me`，仅适合本地首次启动，部署到局域网后应立即替换。

登录后可以看到：

- **更新历史**：查看所有处理批次和结果
- **新建更新**：上传模板和截图
- **产品目录**：查看或导入产品
- **账号**：管理员创建普通用户或其他管理员

### 2. 准备并导入产品目录

管理员进入“产品目录”，上传 UTF-8 编码的 CSV。表头必须完全一致：

```csv
product_name,product_code,product_type
易方达环保主题混合A,001856,public
仁桥金选泽源5B,P001,private
```

字段要求：

| 字段 | 说明 |
| --- | --- |
| `product_name` | 产品名称，需与 Excel 中的名称一致 |
| `product_code` | 产品代码，公募基金通常使用基金代码 |
| `product_type` | 只能填写 `public` 或 `private` |

导入规则：

- 三列缺一不可
- 产品代码不能重复
- 已存在的代码如果名称或类型冲突，整次导入会被拒绝
- 当前网页入口只接受 CSV，不接受 XLSX 产品目录

### 3. 上传一次更新

进入“新建更新”：

1. 在“截止日期”中填写本次数据截止日。默认值为当前日期，可改成实际报告截止日。
2. 上传原始净值模板，例如 `净值更新表.xlsx`。
3. 私募或渠道产品上传一张或多张 PNG/JPEG 高清截图。
4. 点击“上传并生成预览”。

同一批次中的截图必须来自同一个报告日期；不同周度或历史报告请分别新建批次。例如，5 月 8 日至 7 月 10 日的十份周报应建立十个批次，不能混传后用一个截止日处理。

私募和公募可以混在同一张截图中：系统会先按产品名称采用截图指标，并自动分段处理超长图片。截图没有某个产品时，系统优先使用产品目录；若目录没有该产品，只会在公开基金目录中找到唯一的精确或已支持历史名称后补全基金代码。多个候选或未命中时不会猜测，保留原值并提示人工审核。私募产品应尽量提供清晰、完整、包含产品名称或代码及指标的截图。

自动补全的公募代码会保存到产品目录，供以后无截图的批次复用。已完成或已人工处理的历史批次不会被自动重新处理；请新建批次验证新的匹配结果。

### 4. 查看预览、人工审核并处理

预览页会汇总待审核和可直接生成的条目数，并显示 Excel 行号、产品名称、匹配来源、已识别字段数、处理状态和原因。待处理条目可直接进入对应的人工审核位置。

常见状态：

| 状态 | 含义 |
| --- | --- |
| `ready` | 已找到产品且数据完整，可以写入结果 |
| `needs_review` | OCR 置信度不足，需要人工关注 |
| `stale` | 数据过旧或指标数据不足，保留原值并标红 |
| `failed` | 数据源请求或处理失败，保留原值并标红 |
| `completed` | 批次全部完成且没有警告 |
| `completed_with_warnings` | 批次完成，但至少有一行数据过旧、失败或待确认 |

对需要人工确认、数据源失败、数据过旧或任何需要修正的条目，点击“人工审核”。页面默认只显示待处理条目，可切换查看已识别完整条目；审核页会将“需补录”和“已识别（可修改）”指标分开显示，未识别的指标会标为“待补”。

当截图已匹配但私募产品不在目录中，人工审核页会默认“自动创建私募产品”。保存审核时，系统以 Excel 原始产品名称创建稳定内部编号的私募目录项；不会依据 OCR 识别出的名称自动建档。后续同名批次会复用该产品。

审核步骤：

1. 在对应条目中选择确认后的产品。
2. 填写一个或多个指标，并在“审核说明”中记录数据依据。
3. 百分比字段填写 `12.34` 表示 12.34%；夏普比填写普通数值，例如 `1.25`。
4. 点击“保存审核”。条目来源会变为 `manual`，操作会写入审计日志。
5. 返回匹配预览，点击“重新生成结果”，人工审核值会优先于 OCR 和公募数据源。

审核表单中未填写的指标不会用空值覆盖 Excel：系统会保留原单元格并标红。只有填满全部指标的人工审核条目才会显示为 `ready`；部分补录的条目仍会以 `stale` 提醒复核。

`needs_review` 条目仍可选择“跳过”。跳过后不会覆盖原单元格，相关指标会标记为旧值/不可用。

确认后点击“开始处理”。处理完成后，点击“下载结果 Excel”。原始 Excel 永远不会被覆盖。

### 5. 导入和跟踪资本市场核心会议

管理员进入“会议跟踪”，上传 `资本市场核心会议跟踪.xlsx`。当前版本只读取名为“近期会议更新”的工作表，并固定使用第二行作为表头；其他横向对比工作表不会被自动导入。

表头必须包含以下九列：

```text
会议/事件
日期
性质/层级
核心表述
资本市场影响
投研映射
后续跟踪
来源链接
更新时间
```

日期支持单日（例如 `2026-06-06`）和范围（例如 `2026-06-17至2026-06-18`）。未能识别的日期仍会保留原始文本，但不会参与日期筛选。

重复导入同一“会议/事件 + 原始日期”时，系统会更新来源字段，不会覆盖团队补充的公司标签、行业标签、参会状态、纪要、待办和结论。导入完成后，页面会显示本次新建和更新数量，并写入审计日志。

所有已登录用户都可以打开会议详情并维护团队记录：

1. 用逗号或分号填写公司、行业标签。
2. 选择未安排、计划参会、已参会或未参会。
3. 填写会议纪要、后续待办和研究结论后保存。
4. 在列表用关键词、日期范围、性质/层级、公司标签或行业标签筛选。

## 数据和指标规则

### 公募数据

默认使用东方财富公开基金净值接口，根据产品目录中的 `product_code` 查询累计净值。对于目录中没有、且截图未覆盖的产品，系统会从东方财富公开基金目录中尝试唯一名称匹配；只接受精确名称或支持的历史名称描述，绝不按相似名称自动写入。请求超时或返回格式异常时，该产品不会中断整个批次，系统会将该行标记为失败并保留原值。

### 截止日和数据新鲜度

- 周收益使用截止日前后可用的周窗口
- MTD 使用上月末到截止日
- YTD 使用上年末到截止日
- 年度收益计算已完成年度
- 公募净值超过 14 天没有更新时视为过旧
- 私募/渠道数据超过 45 天没有更新时视为过旧

### 输出内容

结果写回固定模板中的指标列，模板的合并单元格、冻结窗格、条件格式和其他非目标内容会保留。过旧、失败或数据不足的指标保留旧值，并使用错误样式提示。

## 多人协作和账号

管理员可以进入“账号”创建普通用户。普通用户可以执行日常上传、预览、人工审核、下载和会议团队记录；产品目录、账号管理和会议 Excel 导入仅管理员可用。

当前页面没有自助注册和密码修改入口。若需要更换初始管理员密码，应在首次启动前修改 `.env`；已经创建的账号需要由后续账号管理功能或数据库运维处理。

## 运维命令

查看服务状态：

```bash
docker compose ps
docker compose exec db pg_isready -U nav -d nav
```

查看日志：

```bash
docker compose logs -f app worker
```

停止服务但保留数据卷：

```bash
docker compose down
```

重新启动：

```bash
docker compose up -d
```

### 备份

备份脚本会同时保存 PostgreSQL dump 和应用文件卷：

```bash
scripts/backup.sh ./backups
```

输出目录中会生成：

- `db-*.sql`：数据库备份
- `files-*.tar.gz`：上传文件和结果文件备份

### 恢复

恢复脚本默认选择备份目录中最新的一组文件：

```bash
scripts/restore.sh ./backups
```

恢复前请确认备份目录正确，并避免在生产服务正在写入时执行恢复操作。

## 故障排查

### 页面打不开

```bash
docker compose ps
curl http://127.0.0.1:8080/healthz
```

如果 `db` 没有显示 `healthy`，先查看：

```bash
docker compose logs db
```

### 登录失败

确认浏览器访问的是部署主机地址，而不是另一台机器的地址。初始化账号只在数据库第一次创建用户时生效；数据库已经存在时，修改 `.env` 不会覆盖已有密码。

### 批次出现 `completed_with_warnings`

打开该批次的预览页，查看每一行的原因。常见原因包括：

- 产品未导入目录或名称不一致
- 公募数据源请求失败
- 截图模糊、字段缺失或 OCR 置信度不足
- 净值数据超过允许的新鲜度

### 结果文件没有更新某些单元格

这是保护性行为：系统在无法确认数据时不会用猜测值覆盖原值，会保留旧值并标红，方便后续人工复核。

## 本地开发

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e '.[dev]'
pytest -q
ruff check .
```

只启动本地 FastAPI（不包含 PostgreSQL）可使用项目默认的 SQLite 配置；完整部署和多人协作请使用 Docker Compose。

## 项目结构

```text
app/
  main.py                  FastAPI 路由和页面
  auth.py                  登录、会话和 CSRF
  catalog.py               产品目录导入
  meetings.py              会议 Excel 导入和来源字段更新
  domain/                  指标计算和精确匹配
  excel/                   XLSX 模板保真更新
  jobs/                    批次和 worker
  ocr/                     OCR 引擎和表格解析
  providers/               公募数据源适配器
migrations/                Alembic 数据库迁移
scripts/                   备份和恢复脚本
tests/                     单元、集成和页面流程测试
```
