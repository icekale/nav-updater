# 人工审核工作台 Design

## Goal

为净值更新批次中的待确认、失败和过旧条目提供可追溯的人工审核闭环。审核人可以重新选择产品、录入或覆盖指标、说明依据，并重新生成结果 Excel。

## Scope

- 在单个更新批次下新增审核页面。
- 展示每一条目的原始产品名称、当前状态、处理来源和失败原因。
- 审核人可选择已激活产品目录中的一个产品，填写任意指标，并填写审核说明。
- 保存后将条目来源标为 `manual`，已填写指标的状态标为 `manual`，未填写指标标为 `stale`。
- 后续处理批次时，人工审核条目优先于 OCR 和公募数据源，并将人工数据写入结果 Excel。
- 每次保存人工审核结果写入审计日志。
- 所有已登录用户可审核，沿用现有批次访问权限和 CSRF 保护。

## Non-goals

- 不引入 Soybean Admin、Vue SPA、独立 API 层或新的登录机制。
- 不提供人工修改原始 Excel 或修改产品目录的快捷入口。
- 不实现多人并发锁定、审核流审批、附件上传或历史版本回滚。
- 不改变现有公募抓取、OCR 和指标计算规则。

## UI

在匹配预览页增加“人工审核”入口，并在批次已经生成过结果时将“开始处理”改为“重新生成结果”。

审核页为服务端渲染表单：每个条目占一行，包含产品下拉框、12 个可选指标输入框、审核说明和保存按钮。页面保留原状态和错误原因，便于对照。指标输入使用百分比数值，Sharpe 使用普通数值，最大回撤使用百分比数值。

审核说明必填。产品选择和至少一个指标也必填，避免保存没有实际结论的审核记录。

## Data Flow

现有 `run_items` 已有 `product_id`、`match_source`、`row_status`、`metric_values`、`metric_status` 和 `error_reason` 字段，不新增数据库迁移。

提交审核表单时：

1. 验证条目属于当前批次、产品处于激活状态、指标为有效十进制数。
2. 把百分比输入转换为小数存入 `metric_values`，Sharpe 原样存入。
3. 把已填写指标状态设置为 `manual`；未填写指标状态设置为 `stale`。
4. 将 `match_source` 设置为 `manual`，将 `row_status` 设为 `ready` 或 `stale`，并记录审核说明。
5. 写入 `audit_logs`。

处理批次时，`match_source == "manual"` 的条目跳过 OCR 和公募查询，直接使用保存的指标。未填写指标进入 Excel 的错误样式集合，原单元格保持不变。

## Error Handling

- 非法产品、空说明、没有任何指标或无法解析的数字返回审核页面并显示错误。
- 没有权限、批次或条目不存在时保持现有 401/404 行为。
- 手工审核只影响当前批次，不改变产品目录或其他批次。

## Testing

- 服务层测试手工审核保存的来源、产品、数值和状态。
- 处理器测试人工数据覆盖外部数据源并保留未填写指标。
- 页面流程测试登录、创建批次、进入审核页、提交审核、重新生成并下载结果。
