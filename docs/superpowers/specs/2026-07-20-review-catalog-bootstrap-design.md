# 人工审核目录补齐 Design

## Goal

让截图已匹配但产品目录缺失的私募产品可在人工审核中完成确认，无需先导入 CSV；让审核人优先处理缺失指标，而不是在完整条目中反复查找。

## Scope

- 审核页默认显示 `needs_review`、`stale` 和 `failed` 条目，并提供查看已识别完整条目的入口。
- 审核页为已有目录匹配的产品自动选中产品下拉框。
- 当当前 Excel 原始产品名没有匹配到激活产品时，产品下拉框提供默认选项“自动创建私募产品”。
- 审核人保存默认选项时，系统以 Excel 原始产品名创建 `private` 产品，并生成稳定的内部唯一编号。
- 已识别指标保留输入值且可编辑；缺失或 stale 指标以“待补”状态突出显示。
- 新建私募产品和审核保存均记录审计日志。

## Non-goals

- 不在 OCR 处理阶段根据 OCR 名称自动创建产品。
- 不修改已有产品名称、代码、类型、别名或启用状态。
- 不提供截图行裁剪、OCR 坐标持久化、产品目录批量编辑或新建产品的手动代码输入。
- 不改变公募产品解析、净值计算、Excel 列结构和登录权限模型。

## Product Identity

自动建档只使用 `run_items.original_values.product_name`，而不是 OCR 输出名称。名称先按现有 `normalize_name()` 规则规范化，再计算 `private-` 加十六进制 SHA-256 摘要前 12 位作为 `product_code`。

保存前按规范化名称检查所有激活产品的当前名称和历史名称：

1. 只有一个产品匹配时，复用该产品，不创建目录项。
2. 没有产品匹配时，创建 `Product(product_name=Excel 原名, product_code=生成代码, product_type="private")`。
3. 多个产品匹配时，不自动选择；审核人必须从下拉框明确选择，且不会创建新产品。

代码碰撞时，保存失败并显示“内部产品编号冲突”，不覆盖现有产品。

## UI

审核页顶栏显示待审核数量和“显示已识别完整条目”的链接。默认列表仅显示需要操作的条目；链接保留同一批次并显示全部条目。

每个审核条目显示原产品名、来源、状态和原因。产品选择规则如下：

- 唯一匹配的激活产品：直接选中。
- 没有匹配：默认选中“自动创建私募产品：{Excel 原名}”。
- 多个匹配：保留“请选择产品”，列出候选和其他激活产品。

指标区保留 12 个输入框。`metric_status` 为 `stale`、`insufficient_data` 或 `failed` 的输入框增加“待补”标签和浅红边框；其他已识别值显示为普通可编辑输入。审核说明仍必填。

## Data Flow

在 `review_response()` 中为每个 `RunItem` 计算产品选择状态，并传给模板。保存路由接收 `product_choice`：现有产品使用 `product:{id}`，自动建档使用 `create_private`。

保存 `create_private` 时，路由调用目录服务按上述规则解析或创建产品，然后调用现有 `save_manual_review()`。创建成功后新增 `AuditLog(action="create_private_product")`，审核仍写入 `AuditLog(action="manual_review")`。

`save_manual_review()`、重新生成和 Excel 写入逻辑保持不变。自动创建的目录项可被后续批次的名称匹配复用。

## Error Handling

- 空产品名不能自动建档，返回审核页错误。
- 内部代码冲突、目录匹配歧义或无效产品选择返回 422，已输入指标和审核说明保留在表单中。
- 批次处理期间继续拒绝审核保存。
- 普通用户可以保存审核和自动建档；目录 CSV 导入和账号管理仍仅管理员可用。

## Testing

- 目录服务测试：规范化名称唯一匹配时复用、无匹配时创建稳定私募代码、多个匹配时拒绝自动选择、代码冲突时失败。
- 页面流程测试：默认过滤待审核条目、完整条目通过查询参数显示、唯一匹配产品被选中、无目录产品的自动建档选项被保存。
- 模板测试：缺失指标有待补标识和样式，已提取指标保留可编辑值。
- 回归测试：保存自动建档后的条目可重新生成结果 Excel，后续相同 Excel 名称批次能复用该产品。
