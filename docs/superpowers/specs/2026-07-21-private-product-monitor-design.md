# 私募产品监控总览设计

日期：2026-07-21

状态：待用户复核

## 目标

为研究员提供一个全中文的“产品监控”总览，集中查看所有启用私募产品的最近业绩、最近更新日期和数据异常。首版优先发现数据缺失或更新过期，减少逐批次翻阅历史记录的时间，并支持将当前筛选结果导出为 Excel。

## 范围

### 产品范围

仅展示产品目录中 `is_active=True` 且 `product_type="private"` 的产品。公募产品和已停用产品不进入监控总览。

每只产品从已有 `RunItem` 与 `UpdateRun` 中取得最新已完成记录：先按 `UpdateRun.cutoff_date` 倒序，再按 `UpdateRun.created_at` 倒序，最后按 `RunItem.id` 倒序。处理中、等待处理和失败后未形成可用结果的批次不作为“最近业绩”来源。

首版不新建业绩快照表，不回填历史数据；直接复用现有产品、批次、审核和指标数据。

### 异常状态

每只产品只显示一个中文状态，按以下优先级判断：

1. **从未更新**：没有任何已完成批次中的关联产品记录；
2. **数据缺失**：最新记录为待人工审核、数据待补或处理失败，或其指标状态中有 `stale`、`failed`、`insufficient_data`；
3. **更新过期**：最新净值截止日严格早于“今天减 10 个自然日”；
4. **正常**：不符合以上任何状态。

`source_blank` 是已确认的空值，不计入数据缺失。若同时存在缺失与过期，优先显示“数据缺失”。

### 产品监控页面

侧边栏新增“产品监控”入口，路由为 `/monitoring`。页面全部使用中文，包括状态、按钮、空态、筛选条件、导出文件名和异常说明。

页面由四部分组成：

1. 顶部概览：私募产品总数、正常数、从未更新数、数据缺失数、更新过期数；
2. 异常优先清单：仅显示从未更新、数据缺失和更新过期产品，按状态优先级、最近截止日从早到晚排序；
3. 全部产品表：产品名称、产品代码、最近净值截止日、最近处理时间、近一周、MTD、YTD、状态和操作；
4. 筛选与导出：按产品名称/代码搜索，按全部、正常、从未更新、数据缺失、更新过期筛选；“导出当前清单”将当前筛选结果下载为 Excel。

表格指标只展示近一周、MTD、YTD，避免横向表格过宽。导出的 Excel 包含这些字段、状态、缺失指标说明及全部现有标准指标，方便后续汇总或转发。

### 跳转与操作

有最新记录的产品，点击“查看批次”进入该记录的匹配预览。状态为数据缺失时，提供“去审核”入口，使用 `show_all=1` 打开该批次的完整审核页面，并定位到该产品条目；从未更新产品不显示无效跳转。

更新过期但指标完整的产品只显示“查看批次”，不强制进入人工审核。

## 数据流

```text
启用私募产品目录
        │
        ├── 无已完成 RunItem ─────────────→ 从未更新
        │
        └── 最新 RunItem + UpdateRun
                    │
                    ├── 待审核 / 缺失指标 ─→ 数据缺失
                    ├── 截止日超过 10 天 ──→ 更新过期
                    └── 其他 ─────────────→ 正常
                                      │
                                      ├── /monitoring 页面
                                      └── 当前筛选结果 Excel
```

状态计算放在独立的监控查询服务中，路由、模板和 Excel 导出共用同一份结果，避免页面与导出出现不同的异常结论。

## 权限与错误处理

- `/monitoring` 和导出接口沿用现有登录保护；
- 无监控产品时显示中文空态，不返回错误；
- 导出时无匹配产品仍生成只有表头的有效 Excel；
- 产品目录、审核记录或批次在读取间被删除时，跳过失效记录，不影响其余产品与导出；
- 不向外部服务发送产品、净值、审核说明或导出数据。

## 验收标准

1. 仅显示启用私募产品；公募和已停用产品均不出现。
2. 最新记录严格按截止日、创建时间、条目 ID 的顺序选择。
3. 从未更新、数据缺失、更新过期和正常四种状态按已确认优先级正确显示；`source_blank` 不会单独触发数据缺失。
4. 异常清单仅显示异常状态，且优先于全产品表展示。
5. 搜索与状态筛选同时影响全产品表、异常清单和导出文件。
6. 导出 Excel 的表头为中文，包含当前表格字段、缺失指标说明和全部标准指标。
7. 数据缺失产品的“去审核”可定位完整审核页；其他状态不出现无效审核链接。
8. 未登录访问页面或导出接口会跳转登录；全量 pytest、Ruff、Unraid 健康检查通过。

## 非目标

- 不在本阶段加入收益大幅波动、回撤阈值或估值异常预警；
- 不增加自选池、邮件/微信提醒、定时任务或消息推送；
- 不新建产品详情页、产品长期收益曲线或历史业绩图；
- 不修改 OCR、人工审核、批量更新或会议跟踪规则；
- 不新增外部数据源或外部 API 调用。
