# 工作台操作与侧边栏布局设计

日期：2026-07-21

状态：待用户复核

## 目标

将投研净值更新工具从顶部导航调整为稳定的工作台侧边栏，并补齐历史批次和账号的安全删除入口。OCR 预览同时区分“数值”“已确认源空值”和“真正未识别”，减少不必要人工审核。

本次坚持服务端渲染和现有表格工作流，不引入前端框架或重新设计数据模型以外的业务模块。

## 视觉方向

重构级别：中度重构。

当前页面结构清晰，但顶部导航与页面内容竞争横向空间，历史和账号页也缺少低频管理操作的清晰边界。新布局采用克制的局域网工作台风格：桌面端为固定左侧导航与宽内容区；使用冷灰背景、白色数据面和少量深蓝选中态；不增加营销卡片、渐变、浮夸阴影或动效。

## 侧边栏与响应式布局

`base.html` 改为应用壳：桌面端左侧导航宽 232px，顶部显示产品名称，中央是主要导航，底部显示当前用户名与退出按钮。导航保留现有权限：账号管理仅管理员可见。当前路由用实心深蓝背景和高对比文字表示；默认项保持安静的文字态。

主内容区占据剩余视口宽度，最大阅读宽度只用于表单和详情页，不限制历史表格、预览表格和会议列表。所有表格保持横向滚动能力。

当视口窄于 800px 时，侧栏变为顶部紧凑导航：品牌、当前页面和可横向滚动的导航项按纵向层级排列，内容区使用 16px 内边距。不会把桌面侧栏挤压成窄列。

## 批次删除

历史页的每一行增加“删除”操作，仅已登录用户可执行。表单包含 CSRF token，并使用浏览器原生二次确认，明确提示该操作不可恢复。

删除规则：

1. 正在处理的批次拒绝删除，并返回可见错误信息；
2. 删除批次关联的 `RunFile`、`RunItem`、原始 Excel、上传截图和结果 Excel；
3. 只允许删除 `DATA_DIR/runs/<run_id>` 内的文件或该批次记录的输出文件，路径必须在数据目录内，防止误删其他文件；
4. 删除 `object_type` 为 `update_run` 或 `run_item` 且属于该批次的审计记录；
5. 写入一条新的“删除批次”审计记录，保留批次编号、删除条目数和文件数，但不保留指标明细；
6. 删除成功后回到历史页并显示一次性提示。

## 账号删除

管理员可在账号管理表中删除其他账号。删除按钮使用同样的二次确认与 CSRF 防护。

规则：

1. 当前登录账号不能删除自己；
2. 系统最后一个管理员不能删除；
3. 被删除账号的历史批次保留，`UpdateRun.operator_id` 改为可空并设为 `NULL`，页面显示“已删除账号”；
4. 该账号的旧审计记录保留，但 `AuditLog.actor_id` 设为 `NULL`，操作上下文不改写；
5. 新增数据库迁移将 `update_runs.operator_id` 设为可空，并在外键删除时使用 `SET NULL`；
6. 管理员删除账号后写入一条独立审计记录。

不提供批量删除、自己删除或“删除账号同时删除历史批次”。

## OCR 预览规则

当前预览中的“已识别 X/12”只统计数值字段，因此已确认的 `source_blank` 被误显示为漏识别。改为在服务端为每个 `RunItem` 计算：

- 数值字段数：`metric_values` 中的有效数值；
- 源空值数：`metric_status` 为 `source_blank` 的字段；
- 已确认数：两者之和；
- 真正未识别字段：`metric_status` 为 `stale` 的字段。

预览显示“已确认 11/12（8 数值＋3 空值）”。原因使用现有指标标签映射，例如 `annual_2023` 显示为 `2023（%）`。

对截图匹配行，唯一产品匹配仍是自动写入前提。确认字段达到 9 项时：

- 12 项均确认：`ready`；
- 9–11 项确认：`partial`，已确认字段写入，未识别字段保留原值并标红；
- 少于 9 项或产品不唯一：`needs_review`。

整体行最低 OCR 置信度不再单独把 9 项以上的行升级为人工审核；在 `partial` 原因中保留“低置信度提示”供研究员查看。这样避免某个短 token 的低分阻塞高覆盖批次。

## 页面细节

- 历史页显示批次、截止日、状态、创建时间、查看与删除操作；删除为低强调危险按钮，不与“新建更新”竞争视觉优先级。
- 账号页的创建表单保持原样，账号表增加操作列；当前账号和最后管理员的删除按钮以说明文字替代，避免无效点击。
- 预览页统计区将“可直接生成”定义为 `ready` 与 `partial` 之和；状态标签用低饱和琥珀色表示 `partial`。
- 原因列优先显示中文缺失字段和非阻塞低置信度提示；不直接显示内部指标键。

## 非目标

- 不删除产品目录、会议记录或公开基金缓存。
- 不新增回收站、定时清理、批量删除或复杂权限系统。
- 不对缺少 OCR token 的字段推断为空值。
- 不改动已存在的 Excel 模板列、结果计算或人工审核保存语义。

## 验收标准

1. 桌面与窄屏均可访问所有原有导航项，管理员权限仍正确生效。
2. 删除已完成批次后，其数据库子记录、批次目录、结果文件和关联批次审计记录均不存在；处理中的批次不会删除。
3. 删除普通账号或非最后管理员后，用户无法登录，历史批次仍可查看并显示“已删除账号”。
4. 当前账号和最后管理员的删除请求返回拒绝，不改变数据库。
5. 预览将 8 个数值加 3 个 `source_blank` 显示为“已确认 11/12”，并将内部字段键翻译为中文标签。
6. 9–11 个确认字段的唯一匹配截图行即使有低置信度 token 也为 `partial`，不计入默认人工审核数；少于 9 个确认字段仍需审核。
7. 单元、集成、端到端测试及 Ruff 继续通过。
