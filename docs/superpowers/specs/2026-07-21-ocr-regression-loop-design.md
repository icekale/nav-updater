# OCR 回归质量闭环设计

日期：2026-07-21  
状态：已确认，待实施计划

## 背景与目标

当前系统已经具备人工审核快照和“质检中心”，能够统计人工确认后的漏识别与值不一致。批次 #12 的 `浑瑾岳桐金选1号B` 暴露了另一类风险：超长截图的切片边界将 `-6.33%` 拆开，旧逻辑把单独识别到的 `-` 错当成“原图为空”。将默认切片改为 1600px 后，重新处理能得到正确的 `-6.33%`。

本功能把一次已确认的错误变成可重复执行的回归案例，并在生产识别中对“空值或漏读”执行一次受控的二次识别。目标是让 OCR 质量可以量化、问题可复核、规则调整可验证，而不是承诺所有截图均可自动识别。

## 已确认的决策

1. 现有历史批次的人工确认结果和已复核的 OCR 错误均纳入回归样本；未人工确认的普通 OCR 输出不能作为标准答案。
2. 回归样本复制原始截图到受保护的专用目录。删除普通更新历史不会删除回归样本。
3. 生产识别遇到疑似空值或字段未读到数值时，先进行二次识别；只有能确认原图为空时才写入 `source_blank`。无法确认时进入人工审核。
4. 已登录用户可以查看质量结果；只有管理员可以导入/提升样本、运行回归验证和管理样本。
5. 新批次保存字段级识别证据。人工审核页展示截图局部、原始文本、置信度和两次识别结果；旧批次没有证据时明确说明，不补造记录。

## 范围

### 1. 独立回归样本库

现有 `OcrReviewSample` 继续是“人工审核前后快照”的审计记录，不承担长期回归资产的生命周期。新增独立回归样本，单个样本对应一个 Excel 产品条目和它的截图资产。

每个回归样本保存：

- 截图副本路径与 SHA-256；相同 SHA 的截图在样本目录中只保留一份物理文件；
- 来源批次/条目的人类可读快照（不是级联删除的业务外键）；
- 产品、Excel 产品名和用于唯一匹配的候选名称；
- 预期产品匹配结果；
- 每个指标的预期值与预期状态（数值、`source_blank` 或待审核）；
- 样本来源（历史人工审核、管理员复核案例）、审核说明、创建人和创建时间；
- 是否启用。禁用样本保留审计与文件，但不进入回归运行。

截图保存在 Docker `app_data` 卷内的 `/data/ocr-quality/samples/`，只通过需要登录的应用接口访问；不提交到 Git，也不发送给外部 OCR 服务。普通历史记录删除时，只删除其原文件和输出 Excel；已提升的样本继续保留。

#### 历史样本导入

管理员在质检中心选择“导入历史已确认样本”后，系统只处理以下记录：

- `OcrReviewSample` 存在人工确认指标，且来源批次中能找到截图；
- 管理员显式提升的已复核 OCR 错误，例如批次 #12 的 MTD `-6.33%`。

如果来源批次有多张截图且无法从现有记录唯一判断图片，导入结果标记为“需要选择截图”，不能猜测。导入过程显示新增、已存在、跳过和待选择数量；重复执行以截图 SHA、产品和预期内容去重，不制造重复样本。

### 2. 生产 OCR 的二次识别

保持现在的首轮策略：RapidOCR 使用 1600px 高度、128px 重叠的分片识别。

当某个指标满足任一条件时触发二次识别：

- 仅识别出孤立的 `-`，可能被当作 `source_blank`；
- 没有得到可解析的指标数值；
- 得到的 OCR 文本无法按当前指标规则落到对应产品行。

二次识别只针对触发条件的截图，使用更密的切片与较大重叠重新读取，再由现有产品匹配和字段解析逻辑按行归并。首轮已经得到的有效值不被二次识别覆盖；二次识别只能填补缺失字段或将“疑似空值”替换为明确的数值。

字段状态遵循以下优先级：

1. 二次识别得到可解析且归属唯一的数值：使用该数值，状态为 `extracted`；
2. 两轮都没有数值，且视觉检测确认是原图破折号/空值：写空，状态为 `source_blank`；
3. 其余情况：不覆盖原 Excel，状态为待人工审核，说明中写明二次识别未解决。

首版不以“低置信度阈值”自动覆盖数值；置信度先用于审核证据与后续分析，避免在缺少标注数据时引入任意阈值。

### 3. OCR 识别证据

新处理批次的每个 OCR 条目在 `RunItem` 保存紧凑的 `ocr_evidence` JSON，按字段记录：

- 来源图片标识和图片 SHA；
- 首轮/二次识别的原始文本、置信度、边界框和分片策略；
- 最终选择的值、状态和选择原因。

审核页基于边界框生成仅登录可看的局部截图。没有可用框时显示对应原图与“没有定位到字段区域”。历史批次不回填证据，以免重新识别后将新结果伪装成历史事实。

### 4. 回归执行与质量中心

质检中心保留现有人工审核质量指标，并新增“回归验证”区域：

- 显示启用样本数、最近一次运行时间、通过数、失败数和未执行数；
- 管理员可手动运行一次回归验证；应用后台按样本逐个执行，避免占用网页请求；
- 每次运行保存运行摘要和按样本的结果（通过、产品未匹配、数值缺失、值不一致、状态不一致或执行失败）；
- 失败项展示预期与实际值，并提供产品、样本和原图证据入口；
- 验证流程只读 OCR 与样本，绝不修改生产 `UpdateRun`、`RunItem`、产品目录或 Excel。

管理员还可从同页执行历史样本导入、显式提升错误案例、启用/禁用样本。普通用户看到只读的最近运行摘要和失败清单。

## 数据流

```text
历史人工审核 / 管理员复核案例
             │
             ├─ 复制截图到受保护样本目录
             └─ 写入不可变回归样本（标准答案）

生产截图 ──→ 首轮 OCR（1600px）
                  │
          缺失 / 孤立 "-" / 无法归行？
                  │ 是
                  ▼
             二次密集 OCR
                  │
     有唯一数值 → extracted；确认空值 → source_blank；其余 → 人工审核
                  │
                  └─ 保存字段级识别证据

管理员运行回归验证 ──→ 对样本重放当前 OCR 流程 ──→ 保存只读结果 ──→ 质检中心
```

## 错误处理与安全

- OCR、复制文件或数据库写入失败时，不创建半成品样本；导入结果必须说明失败原因。
- 回归单条样本失败不阻断其余样本；运行结果记录该样本的错误，但整个运行仍能给出完整摘要。
- 不存在或校验值不符的样本图片标记为“样本文件不可用”，不能静默跳过。
- 多图来源的样本在未明确选择源图前不得自动导入。
- 截图、指标、审核说明和回归结果只保留在局域网 Docker 卷；所有查看图片、运行验证和样本管理操作记录审计日志。

## 测试与验收

1. 将批次 #12 的截图和 `浑瑾岳桐金选1号B` MTD `-0.0633` 提升为样本，当前 RapidOCR 规则的回归结果必须通过。
2. 将该样本的预期值改错或模拟 `source_blank`，回归运行必须报告“值不一致”或“状态不一致”。
3. 仅识别到 `-` 的模拟输入必须触发二次识别；二次识别读到 `-6.33` 时最终状态为 `extracted`，不得为 `source_blank`。
4. 两轮都无法确认的字段必须进入人工审核；确认原图为空的字段不进入审核。
5. 回归运行前后，生产批次、产品目录和生成 Excel 的数据均不改变。
6. 删除来源更新历史后，已提升样本仍可执行回归；图片校验值一致。
7. 权限测试覆盖：未登录无法访问；普通用户无法启动运行、导入、提升或管理样本；管理员可执行。
8. 新批次审核页展示两轮 OCR 证据；旧批次明确缺少证据。
9. 全量 pytest、Ruff、Docker 构建、Unraid 健康检查和一次生产回归运行均通过。

## 非目标

- 不训练、微调或替换 RapidOCR 模型；
- 不自动批量重处理历史更新；
- 不将私募截图发送给第三方服务；
- 不把回归验证作为用户上传时的同步阻塞步骤；
- 不凭 OCR 置信度自动修改已有数值。

## 与既有质检中心的关系

本设计扩展 `2026-07-21-ocr-quality-loop-design.md` 已实现的人工审核质量统计。若两份设计存在冲突，以本设计为准，具体调整为：回归样本独立于普通批次删除生命周期；`source_blank` 先经过二次识别再确认；质检中心增加可执行的回归验证、受保护样本与字段证据。
