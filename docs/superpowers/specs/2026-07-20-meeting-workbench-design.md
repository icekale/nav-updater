# 资本市场核心会议工作台 Design

## Goal

将 `资本市场核心会议跟踪.xlsx` 中“近期会议更新”工作表转为局域网应用中的可检索会议库，并允许团队成员补充参会、纪要、待办和结论。

## Scope

- 管理员上传 `.xlsx` 并导入“近期会议更新”工作表。
- 读取会议/事件、日期、性质/层级、核心表述、资本市场影响、投研映射、后续跟踪、来源链接和更新时间九列。
- 解析单日和 `YYYY-MM-DD至YYYY-MM-DD` 日期范围，用于筛选。
- 按会议名称和原始日期生成稳定来源键，重复导入时更新来源字段，不覆盖人工字段。
- 提供会议列表，按关键词、开始/结束日期、性质/层级、公司标签和行业标签筛选。
- 提供会议详情页，所有已登录用户可维护公司标签、行业标签、参会状态、纪要、待办和结论。
- 记录导入和人工维护操作的审计日志。

## Non-goals

- 不自动导入“五年规划”“中央经济工作会议”等横向对比矩阵。
- 不提供会议自动采集、AI 摘要、提醒或任务分派。
- 不引入 Soybean Admin、Vue SPA、Redis 或新的鉴权体系。
- 不把公司和行业做成主数据表；本期使用可搜索的自由文本标签。

## Data Model

新增 `meetings` 表，包含来源字段、标准化的 `date_start`/`date_end`、来源键和人工字段。来源字段包括会议名称、原始日期、性质/层级、核心表述、资本市场影响、投研映射、后续跟踪、来源链接、更新时间。人工字段包括公司标签、行业标签、参会状态、纪要、待办和结论。

`source_key` 唯一，值由规范化会议名称与原始日期组成。导入更新来源字段和导入时间，但不会清空人工字段。会议没有附件、子任务表或多级审批状态。

## UI

顶栏新增“会议跟踪”。会议列表显示日期、会议名称、性质/层级、资本市场影响摘要、公司/行业标签、参会状态和详情入口。管理员可打开导入面板上传 Excel。

详情页按“来源内容”和“团队记录”两段展示：前者只读，后者为保存表单。标签使用逗号或分号分隔的文本，便于直接筛选。参会状态为“未安排、计划参会、已参会、未参会”之一。

## Import Flow

导入器定位名为“近期会议更新”的工作表，读取第二行表头。缺少任一来源列则拒绝导入并显示错误。空会议名称行跳过。单条日期不可解析时仍导入原始日期，但日期筛选不可命中；该条会标记为未标准化日期。

上传完成后返回导入数量及更新数量。导入操作仅管理员可用，并记录上传文件名、导入/更新数量和操作者。

## Security and Errors

- 延续现有 HttpOnly 会话和 CSRF 表单保护。
- 导入仅管理员可用；会议详情和团队记录对已登录用户开放。
- 手工保存必须使用允许的参会状态；其他字段允许为空。
- 会议不存在返回 404，非法工作簿或表头返回表单错误且不部分写入。

## Testing

- 日期范围解析、表头验证和来源键生成的单元测试。
- 导入服务测试：新建、重复更新以及保留已有人工字段。
- 页面流程测试：管理员导入 Excel、按条件筛选、普通用户填写团队记录、审计日志落库。
