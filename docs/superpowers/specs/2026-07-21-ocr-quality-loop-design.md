# OCR 质检闭环设计

日期：2026-07-21

状态：待用户复核

## 目标

让研究员能够量化截图 OCR 的实际质量，并把已完成的人工审核自动沉淀为可信的质检样本。工作台提供一个独立的“质检中心”，用于发现高频漏识别字段、低质量产品和仍待处理的批次；从问题列表可直接进入对应人工审核页。

首版选用 A「质检中心」布局：质量概览、问题清单与审核入口集中在同一页面，不改变现有上传、预览、人工审核和 Excel 导出流程。

## 已确认的业务规则

1. 研究员保存人工审核时，系统自动沉淀样本，不增加“是否记录样本”的额外步骤。
2. 历史人工审核已覆盖原 OCR 值，无法可靠回算准确率；可信准确率从本功能上线后的新审核开始计算。
3. 截图中已识别为 `source_blank` 的字段仍按当前规则直接写空且不进入人工审核。首版仅把其作为运营计数，不将其纳入人工确认后的 OCR 准确率分母。
4. 批次被永久删除时，其质检样本一并删除，保持“永久删除关联记录”的既有语义。

## 范围

### 质检样本

新增 `ocr_review_samples`。每次保存一个来源为截图或未匹配截图的人工审核条目时，在覆盖 `RunItem` 当前值之前写入一条不可变样本。

每条样本保存：

- `run_id`、`run_item_id`、审核人、保存时间；
- 审核前的产品匹配结果、指标值与指标状态（OCR 快照）；
- 审核后确认的产品、指标值与指标状态（人工确认快照）；
- Excel 原始产品名称和审核说明，用于定位和解释；
- 审核版本号。若同一条目再次审核，保留新版本；统计时仅使用该条目的最新版本，避免重复计数。

截图来源包括 `image` 和 `none`：前者用于衡量已匹配截图的识别质量，后者用于统计“截图未找到对应产品/字段”的漏识别。`public_provider` 来源不生成 OCR 质检样本，避免把公开基金接口结果误算为 OCR 表现。

人工确认中实际填写或保留在表单内提交的数值为该字段的确认答案。未提交的字段记为“未确认”，不进入字段准确率分母；这样不会把“暂未补齐”误当成错误 OCR 或已确认空值。

### 质量指标

新页面 `/quality` 默认查看最近 30 天、各条目的最新人工确认版本，并显示：

- 字段一致率：有人工确认的字段中，OCR 数值与人工确认值相同的比例；
- 漏识别数：人工确认有值、而 OCR 没有提取该字段的数量；
- 值不一致数：OCR 提取了数值但与人工确认值不同的数量；
- 待审核条目数：当前仍处于待审核状态的 `RunItem` 数；
- `source_blank` 计数：仅作运营观察，与字段一致率独立展示；
- 产品匹配情况：审核前产品匹配与人工确认产品一致、未匹配或被改正的数量。

数值比较使用系统已有的标准化内部值（百分比均为小数），不比较输入展示格式，避免 `1.2%` 与 `1.20%` 被判为不同。

### 质检中心页面

页面沿用现有侧边栏和紧凑表格风格，增加“质检中心”导航入口。页面由三部分组成：

1. 顶部概览：最近 30 天的字段一致率、待审核数、漏识别数和 `source_blank` 数；
2. 高频问题：按指标字段聚合，显示人工确认样本量、一致率、漏识别数和值不一致数；
3. 需关注产品：按人工确认产品聚合，显示相同指标，并列出最近问题条目。

每个最近问题条目提供“去审核”链接，目标为 `/updates/{run_id}/review#review-item-{item_id}`。已删除批次的样本会级联删除，不会产生失效链接。

## 数据流

```text
截图 OCR → RunItem 当前识别值
                    │
人工审核保存 ───────┼─→ 先写 OCR 质检样本（原值 + 人工确认值）
                    │
                    └─→ 按现有规则更新 RunItem / 重新生成 Excel

OCR 质检样本（每条目取最新审核版本）
                    │
                    └─→ /quality：概览、字段问题、产品问题、跳转审核
```

人工审核、质检样本和 `RunItem` 的写入使用同一个数据库事务：审核或样本任一保存失败时，两者都不提交。样本中保存审核时刻的快照，不依赖后续会被覆盖的 `RunItem.metric_values`。

## 错误处理与权限

- `processing` 批次继续使用现有规则拒绝人工审核，因此不会产生不完整样本；
- 样本写入失败时，向研究员显示保存失败，原审核内容不部分提交；
- `/quality` 需要现有登录态，不向外部服务发送截图、净值、审核说明或样本数据；
- 质检页面只读，不在页面上修改样本、OCR 设置或历史记录。

## 验收标准

1. 对截图来源条目保存人工审核后，数据库包含原 OCR 快照和人工确认快照，且 `RunItem` 仍按原有流程更新。
2. 审核保存失败时，不创建质检样本，也不更新 `RunItem`。
3. 同一条目重复审核时保留多个版本；质量统计只计算最新版本一次。
4. `public_provider` 条目保存审核后不进入 OCR 质量统计。
5. `/quality` 正确计算字段一致、漏识别、值不一致、待审核和 `source_blank` 指标，并可按字段与产品显示聚合结果。
6. 最近问题条目可跳转到对应审核位置；删除批次时，关联质检样本不残留。
7. 全量 pytest、Ruff 和 Unraid 健康检查继续通过。

## 非目标

- 不回填或猜测历史审核的 OCR 准确率；
- 不训练或微调 OCR 模型，不自动修改 OCR 参数；
- 不新增第三方 OCR 服务、外部数据传输或模型成本；
- 不改变 `source_blank` 的自动留空策略；
- 不新增跨日期筛选、导出质检报告、图片标注工具或人工样本编辑器。
